# FDE MENTAL MODELS — DAY 1 APPLICATION

## Purpose

Apply the programme's mental models to your Day 1 domain analysis, demonstrating
that you hold them actively rather than as definitions.

## Required Headings

1. Build an agent or not — how you used this model today
2. Blast radius minimisation — the first blast radius you identified and what it told you
3. "Who must approve this" — the approval chain you mapped before writing any design
4. One model you expect will be most important to you on Day 14 and why (one paragraph)

## Guiding Questions

- Which model changed a decision you had already made before it occurred to you?
- Which model was hardest to apply today and why?
- If a colleague read only this document, would they understand how you used each model — or just that you know the definition?

## Acceptance Criteria

- Each model entry references a specific decision or observation from today's work (not the definition)
- Blast radius entry names a downstream system or process (not just "things could break")
- Approval chain entry names a role, not a team or department
- Day 14 prediction is grounded in something you observed today, not in the syllabus description
