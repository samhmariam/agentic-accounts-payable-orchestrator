# AegisAP Codification Bridges

These bridge notes are the Portal -> Notebook -> Production Rosetta Stone for
the curriculum.

Use them when a learner can find the broken Azure state but has not yet turned
that observation into durable code.

## Translation Cycle

1. Incident: the system is broken.
2. Portal Investigation: locate the broken state.
3. Lab Repair: reproduce and prove the fix interactively.
4. Codification Bridge: map the portal observation to the exact repo change.
5. Production Patch: make the permanent code or IaC edit.
6. Verification: prove tests, artifacts, and cloud state agree.
